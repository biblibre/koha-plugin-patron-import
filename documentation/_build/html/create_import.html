Créer un nouvel import

Une fois le plugin installé, accédez à sa configuration depuis la page d’accueil des plugins Koha.

Trouver le plugin Patron import dans la table et cliquer sur Action->configure. Ici vous pourrez ajouter un ou plusieurs imports et voir les existants. Cliquer sur « New import » remplir le formulaire.

Le nom de l’import peut être ce que vous voulez. Il y a 2 options pour le type de source: CSV ou LDAP server. Tous les autres éléments sont indiqués.

Pour créer un nouvel import, cliquer sur « New import » sur la page de configuration et compléter le formulaire.

  • Import name: Nom de l’import. Peut être ce que vous voulez.

  • Flow type: Type de données source. Cela peut provenir d’un CSV ou d’un serveur LDAP.

  • Create only: Si cette case est cochée, seulement les nouveaux utilisateurs sont crées, il n’y a pas de mise à jour.

  • Auto Cardnumber: Laisser le plugin créer les numéros de carte pour les adhérents.

  • Send email to new patrons: A la manière de la fonctionnalité incluse dans Koha sur les imports CSV, envoyer un mail d’après le modèle “WELCOME”

  • Clear reports and logs older than: Nombre de jours de stockage des rapports dans la base de données.Ensuite ils seront supprimés.

Réglages CSV

  • CSV path: Chemin du fichier CSV a importé. S’il est vide, vous devez le spécifier comme paramètre du script d’import.

  • Binary: option CSV, voir https://metacpan.org/pod/Text::CSV#binary

  • End of line: Option CSV, voir https://metacpan.org/pod/Text::CSV#eol

  • Char separator: Option CSV, voir https://metacpan.org/pod/Text::CSV#sep_char

  • Quote char: Option CSV, voir https://metacpan.org/pod/Text::CSV#quote_char

  • Empty is undef: Option CSV, voir https://metacpan.org/pod/Text::CSV#empty_is_undef

  • Allow loose quotes: Option CSV, voir https://metacpan.org/pod/Text::CSV#allow_loose_quotes

Gestionnaire de configuration

Vous pouvez exporter votre configuration en cliquant sur le lien « Export this configuration ». Quand vous cliquez dessus, une entrée vous invitant à choisir le nom de votre fichier YAML file apparaît.

Vous pouvez aussi appliquer une configuration depuis un fichier YAML (bien structuré). sur un import existant ou sur un nouvel import directement.
