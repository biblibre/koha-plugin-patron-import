Valeurs par défaut

Les valeurs par défaut évitent qu’un champ soit laissé vide et le force à contenir une valeur dans Koha.

A ne pas confondre avec la correspondance “If none of these rules match” dans la table de correspondance de valeurs, qui s’applique même si le champ contient une valeur.Les valeurs par défaut sont seulement définies si le champ est vide ou n’existe pas dans la source.

Pour ajouter une valeur par défaut à un champ, dans la page correspondante, sélectionner un champ (Koha), saisir la valeur par défaut désirée et cliquer sur le bouton « Add ».

Avertissement

Les valeurs par défaut ne sont pas possibles pour les attributs étendus.

Important

A noter que les champs categorycode et branchcode ont un comportement particulier avec les valeurs par défaut. Durant l’import, ces champs sont supprimés si la valeur ne correspond pas à une catégorie ou une branche valides dans Koha. Donc si vous choisissez des valeurs par défaut pour ces champs, elles seront appliquées même s’ils ne sont pas vides.
